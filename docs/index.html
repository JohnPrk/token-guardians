The Problem

토큰이 왜 이렇게 빨리 사라지는지 혹시 아시나요?

Claude는 지난 대화를 따로 기억해 두지 않아요. 그래서 메시지를 보낼 때마다 그동안 쌓인 대화 전체를 다시 입력으로 받아 처리하고, 턴이 쌓일수록 한 번에 드는 토큰도 선형으로 불어납니다.

출처 · 컨텍스트 윈도우 · Claude Docs

그래서 '프롬프트 캐싱'이라는 게 있어요. 같은 앞부분을 짧은 시간 안에 다시 보내면 이미 처리해 둔 부분은 다시 계산하지 않고 이어 쓰는데, 이 캐시는 기본 5분 동안 유지되고 쓸 때마다 5분이 다시 채워집니다.

출처 · 프롬프트 캐싱 · Claude Docs

이렇게 캐시가 히트하면 기존에 보냈던 메시지들은 토큰을 1/10만 쓰게 돼요. 문제는 그 5분이 워낙 짧아서, 다른 작업을 진행하다가 깜빡하기 쉽다는 점이에요.

Setup

3단계면 끝

처음 한 번만 등록하면 30초마다 자동으로 갱신돼요.

1

캐릭터 고르기

데스크톱 모서리에 앉을 친구를 고릅니다 (판다 · 고양이 · 햄스터). 나중에 설정에서 변경 가능.

2

서비스 선택 — Claude 또는 Gemini

새 계정 추가에서 Claude 또는 Gemini 탭을 고릅니다. 두 서비스 모두 세션 쿠키 한 줄로 연동.

3

쿠키 붙여넣기

안내대로 DevTools → Network 탭에서 cookie: 한 줄을 복사해 붙여넣으면 끝. 테스트 버튼으로 즉시 확인. (Claude는 자동 가져오기도 지원)
